# Support

## Getting help

Use GitHub Issues for:

- bug reports
- feature requests
- documentation improvements

When opening an issue, include:

- Elemta commit/version
- deployment type (local/docker/k8s)
- config excerpt (sanitize secrets)
- logs/error output
- exact reproduction steps

## Security issues

For sensitive vulnerabilities, do not open public issues.

See [SECURITY.md](SECURITY.md).
